# ワールド作成システム仕様書

## 1. 概要

ユーザーが3Dワールドをゲーム内で作成・公開できるシステム。
ワールドは**地形**と**オブジェクト**の 2 要素で構成され、ギミックで動的な演出を追加できる。

UI 仕様詳細: `docs/screens-and-modes.md` セクション 11 参照。

---

## 2. ワールドスロット

| ユーザー種別 | ワールドスロット |
|---|---|
| 通常ユーザー | 5個 |
| プレミアム会員 | 50個 |

- スロットを消費してワールドを作成する
- ワールドを削除するとスロットが解放される
- プレミアム解約時: 6個目以降のワールドはロード不可（データ削除なし）

---

## 3. ワールドオブジェクト

### 3.1 フォーマット

- ファイル形式: GLB（`com.unity.cloud.gltfast` で読み込み）
- テクスチャ: Diffuse 1枚のみ（サイズは Section 4 参照）
- マテリアル: カスタム Unlit シェーダー（アバター・アクセサリと同一）

### 3.2 種別

| 種別 | 入手方法 | 費用 |
|---|---|---|
| デフォルトオブジェクト | 最初から使用可能 | 無料 |
| ショップオブジェクト | ショップで購入 | コイン |
| マイオブジェクト | ユーザーがGLBをアップロード | スロット消費（無料） |

- デフォルトオブジェクトは管理画面から登録する
- 購入したオブジェクトはユーザーアカウントに紐づき、全スロットのワールドで使用可能
- マイオブジェクトはアップロード後すぐに自分の全ワールドで使用可能（詳細: セクション 3.8）

### 3.3 コライダー仕様

各オブジェクトには幅（X）・高さ（Y）・奥行（Z）のコライダー寸法が設定される。

**サイズ指定:** 0（装飾オブジェクト）または 0.25m 以上の 0.25m 単位の値（上限なし）

| 判定 | 条件 | コライダー |
|---|---|---|
| 装飾オブジェクト | W = D = H = 0 | なし |
| 物理オブジェクト | いずれかの寸法 > 0 | 軸平行ボックスコライダー |

- 物理コライダーはグリッドに整列（ビジュアルメッシュの回転に追従しない）
- **コライダーがビジュアルスケールを決定する**（コライダーサイズを変更するとメッシュがそれに追従してスケールする）
- ワールドエディタの拡大縮小モードではインスタンスごとのサイズ上書きが可能（セクション 3.5 参照）

### 3.4 グリッド・配置仕様

- グリッド単位: 0.5m
- グリッドサイズ: 63 × 31 × 63 グリッド（= 31.5m × 15.5m × 31.5m の空間、Y up）
- 原点: ワールド原点 (0, 0, 0) にグリッドインデックス (0, 0, 0) が一致する
  - グリッドセルはインデックスを中心とするため、境界線はグリッド単位の半分（0.25m）ずれた位置になる
  - 例: 正の X 方向に最初の境界線は X = 0.25m、2 番目は X = 0.75m
  - X・Z は ±15.75m まで、Y は ±7.75m まで（中心 = 原点）
- 配置基準点: オブジェクトのピボット（底面中心）をグリッドセル中心に合わせる

**プレイヤーリスポーン境界:**

- ワールドのプレイ可能範囲は原点中心の 64m × 64m × 64m の空間
- この範囲より外に出たプレイヤーはスポーン位置に自動リスポーンする

### 3.5 変形操作（移動・回転・拡大縮小）

ワールドエディタのツールバーで操作モードを切り替える。

#### 移動モード

- グリッドにスナップして移動（0.5m 単位）

#### 回転モード

- Y 軸（水平）のみ 45 度刻みで回転（0° / 45° / 90° / 135° / 180° / 225° / 270° / 315°）
- コライダーは軸平行のまま（回転しない）

#### 拡大縮小モード

- 各軸（W / D / H）を 0.25m 単位で独立して増減する
- 最小サイズ: 0.25m / 軸（装飾オブジェクトを除く）
- コライダーがビジュアルスケールを決定する（コライダーサイズ変更 → メッシュ追従）
- 装飾オブジェクト（コライダーなし）はビジュアルのみスケール（コライダー変更なし）
- 変更はインスタンスレベル（配置単位）で world JSON に保存される（セクション 12.2 参照）
- アスペクト比変更を許容する

**スケールロック（ショップオブジェクト）:**

- ショップオブジェクトには管理画面でスケールロックフラグを設定できる（デフォルト: ロックなし）
- ロック時: 拡大縮小モードで選択しても操作不可（ロックアイコン表示）
- デフォルト・マイオブジェクトにはスケールロックなし

### 3.6 オブジェクト数上限

| 種別 | 上限 |
|---|---|
| オブジェクト合計 | 400個 |

- ギミックで**非表示**のオブジェクトはカウント対象外
- 特殊オブジェクト（スポーン・ポータル・エリア）はカウント対象外

### 3.7 テクスチャ改変

#### ワールドスコープカスタマイズ（基本）

- ワールドごとにオブジェクト種別ごとのテクスチャを **1 種類** カスタマイズできる
- カスタマイズはそのワールドにのみ適用され、他のワールドには影響しない
- カスタマイズはワールド定義 JSON に格納され、ワールド保存時のアトラス生成に使用される
- 常に編集可能（Edit OK / NG フラグなし）

#### 保存バリアント（カスタマイズスロット）

- ワールドスコープのカスタマイズを **保存バリアント** としてスロットに保存できる
- 保存バリアントはそのワールドを離れても保持され、任意のワールドで使用できる
- スロット数: 通常ユーザー **10** / プレミアム会員 **100**
- プレミアム解約時: 11 個目以降のスロットはロック（データ削除なし）
- スロット上限に達している場合: 既存バリアントを削除してから保存
- 保存バリアントはオブジェクトパレット内「保存バリアント」タブから選択・配置できる
- 保存バリアントのテクスチャは固定（ワールドで配置してもそのワールドで再改変はできない）

#### テクスチャコストとの関係

- ワールドスコープカスタマイズ: 元の objectTypeId と同一エントリとしてコストを計算
- 保存バリアント: それぞれ独立したテクスチャエントリとして別にコストを計算

#### 透明ピクセル

アバター・アクセサリと同様に、ワールドオブジェクトのテクスチャにも透明ピクセルを使用できる。テクスチャ編集時にベースレイヤーを編集することで不透明・透明を制御する（詳細: `docs/screens-and-modes.md` セクション 21.9 参照）。

#### 仕様詳細

仕様詳細: `docs/unity-game-abstract.md` セクション 8 参照

### 3.8 マイワールドオブジェクト（ユーザーアップロード）

ユーザーが自分専用のワールドオブジェクトとして GLB ファイルをアップロードできる仕組み。

#### ファイル仕様

| 項目 | 値 |
|---|---|
| ファイル形式 | GLB |
| ポリゴン数 | 最大 256 tris |
| テクスチャ | Diffuse 1枚のみ・16/32/64/128/256px 正方形 |
| ファイルサイズ | 200KB 以内 |
| スケール | 1 unit = 1m |
| ピボット | 底面中心 |

#### 制約・運用

- コライダーサイズはアップロード時に GLB の AABB（軸平行バウンディングボックス）から自動算出し 0.25m 単位に切り上げる。全軸 0 のモデルは装飾オブジェクトとして自動判定
- アップロード後はコライダータブ（セクション 13）から調整可能
- スロット: 通常10個 / プレミアム100個（プレミアム解約時はロック・データ保持）
- 用途: 自分のワールド専用。他ユーザーのワールドには配置不可
- 配置済みのマイオブジェクトはワールド訪問者にも表示される。GLB はアップロード者本人には永続キャッシュ・訪問者には一時キャッシュとして配信
- テクスチャ改変・保存バリアント作成可（他オブジェクト種別と同様）
- 検証は API（Go）側で直接実施（ポリゴン数・容量・テクスチャサイズ）
- モデレーション: 管理画面からBAN可能（BAN後は非表示・スロット解放なし）

UI 仕様: `docs/screens-and-modes.md` セクション 12 参照

### 3.9 プレミアム限定デフォルトオブジェクト

プレミアム会員のみ配置できる特別なデフォルトオブジェクト。

#### 数字オブジェクト

- **種別**: 装飾オブジェクト（コライダーなし・W=D=H=0）
- **対象**: プレミアム会員のみ
- **配置上限**: 1 ワールドにつき **30 個**まで（通常のオブジェクト合計数 400 にも含まれる）
- **表示内容**: 以下のいずれかを指定して数値を表示する。数字以外の文字は表示不可

| 参照元 | 説明 |
|---|---|
| ワールドステートの値 | 指定ステート番号の現在値をリアルタイム表示 |
| プレイヤーのステートの値 | ルーム参加 X 番目のプレイヤーの指定ステートの値をリアルタイム表示 |
| 固定値 | 変化しない整数値を表示 |

- ステートが更新されると、そのステートを参照している数字オブジェクトの表示も即座に更新される

## 4. テクスチャコストシステム

### 4.1 コスト計算

- コスト上限: **4,096**
- カウント対象: ワールドで使用する一意の objectTypeId および savedVariantId ごとに 1 回
  - objectTypeId をワールドスコープカスタマイズした場合: カスタム後のテクスチャサイズでコストを計算
  - 保存バリアントは objectTypeId とは別エントリとしてそれぞれカウント
- コストが上限に達したとき: それ以上の objectTypeId / savedVariantId を追加不可

### 4.3 コスト上限超過時の挙動

**編集中（未公開）:**

- 既に配置済みのオブジェクトはそのまま移動・削除・保存が可能
- 新規オブジェクトの追加のみ不可（追加しようとするとエラーメッセージを表示）
- プレミアム解約などでコスト上限が引き下げられた場合も同様

**ワールド公開時（公開バージョン更新時）:**

- バリデーションでコスト上限チェックを実施し、超過している場合は公開を拒否する
- 既に公開済みのバージョンはそのまま維持される（他のプレイヤーは引き続き遊べる）
- 超過状態が解消されるまで新バージョンの公開・更新はできない

**公開バージョンのアトラスについて:**

- 公開バージョンは公開時点のアトラスを使用する
- ワールドオブジェクトのテクスチャ解像度を後から変更しても、公開バージョンを更新しない限り変更前の解像度のアトラスがそのまま使われる

### 4.2 テクスチャサイズとコスト

| テクスチャサイズ | コスト | 単一種別のみ使用時の最大種類数 |
|---|---|---|
| 16 × 16 | 1 | 4,096 |
| 32 × 32 | 4 | 1,024 |
| 64 × 64 | 16 | 256 |
| 128 × 128 | 64 | 64 |
| 256 × 256 | 256 | 16 |
| 512 × 512 | 1,024 | 4 |

コストの計算式: `(サイズ / 16)²`（例: 64px → (64/16)² = 16）

コスト上限 4,096 に対応するアトラス面積: 4,096 × 256 ピクセル = 1,048,576 ピクセル = 1024 × 1024。コスト上限以内のテクスチャはすべて 1024 × 1024 のアトラスに収まることが保証される（セクション 5 参照）。

### 4.3 ギミックとコストの関係

| 状況 | コストの扱い |
|---|---|
| ギミックで非表示のオブジェクト種別 | コスト対象外 |
| ギミックで種類切り替え（A → B）のオブジェクト | A・B 両方のコストを合算 |

### 4.4 コスト表示

- ワールドエディタ内でリアルタイムにコスト使用率を表示
- 表示形式: `使用コスト / 4,096`（例: 2,048 / 4,096 ≒ 50%）またはプログレスバー

## 5. ワールドアトラスシステム

- ワールド保存時にサーバー側でアトラステクスチャを生成（固定: 1024 × 1024）
- 使用中の全オブジェクトテクスチャをコストの大きいもの（サイズの大きいもの）から順に貪欲パックする
  - コスト上限（4,096）以内であれば 1024 × 1024 に必ず収まることが保証される
- 各オブジェクトのメッシュ UV をアトラス座標に変換する情報をワールド定義 JSON に記録
- 実行時: 1 マテリアル・1 テクスチャで全オブジェクトを描画（アバターアトラスと同方針）

## 6. ワールドタグ

### 6.1 タグ仕様

- ユーザーが自由にテキストを入力するフリーフォーム形式
- 1ワールドにつき最大 **5個**
- タグは保存前にサーバー側で正規化する（小文字化・全角半角統一・前後トリム・Unicode NFC）
  - 正規化後の文字列を `tag_normalized` カラムに保存し、検索・重複排除はこちらを使用
  - 表示用の元テキスト（ユーザー入力まま）も保持する
- 1タグの最大文字数: 20文字
- 禁止タグ（管理画面で登録した BAN リスト）はサーバー側で登録・取得時に除外
- タグ数の上限 / BAN チェックはサーバー側（API）で実施

### 6.2 タグ検索

- 検索はタグの `tag_normalized` に対して完全一致で実施（部分一致は不可）
- 複数タグ指定時は **AND** 条件（指定タグをすべて含むワールドを返す）
- ワールド名検索とタグ検索は同時に指定可能

### 6.3 管理画面でのタグ管理

- BANタグ登録: 管理画面から正規化済み文字列を BAN リストに追加
- BAN 後: 既存ワールドの当該タグも取得時に除外（データ削除なし）
- タグ使用数の集計（ワールド数）を管理画面で確認可能

## 8. 特殊オブジェクト

特殊オブジェクトはオブジェクト数カウント・テクスチャコストの対象外。
ワールド作成モードでは補助表示（実行時は不可視）。

| 種別 | 内容 | 制約 |
|---|---|---|
| スポーン位置 | プレイヤーがワールドに入場するときの初期出現位置 | 1 ワールドにつき正確に 1 つ必要（ワールド公開の必須条件） |
| エリア | ギミック「エリア侵入」イベント用の不可視ゾーン。ボックス型でコライダーサイズと同じ選択肢を使用 | 数の制限なし |
| ルーム内入口ポータル（白） | 触れるとペアの出口ポータルへワープ。転送先の出口ポータルを設定する | ルーム内ポータル合計 8 つ以内。全入口に出口の設定が必要（公開必須条件） |
| ルーム内出口ポータル（黒） | 入口ポータルから転送されてきたプレイヤーが出現する地点 | ルーム内ポータル合計 8 つ以内 |
| ワールドポータル | 設定した別ワールドのスポーン位置へ移動（現在のルームから退室）。ポータル領域中央にそのワールドのサムネイルを表示 | 最大 4 つ。目的地ワールド ID を設定（公開済みワールドのみ選択可） |

**スポーン位置・ポータルの重複ルール:**

スポーン位置・ポータルは他のオブジェクトや地形と重複した状態で配置は可能だが、重複している間は警告フラッシュメッセージが表示され、ワールドのプレイ・公開ができない。

**スポーン位置コライダー仕様:**

- サイズ: 1m × 1.5m × 1m（Y up）
- スポーン原点: コライダーの (0.5, 0, 0.5)（XZ 中央・Y 最下部）
- 削除・回転・拡大縮小は不可

## 9. ギミックシステム

### 9.1 状態（State）仕様

**ワールド状態:**

- 1 ワールドにつき最大 10 種類
- 各状態: 0 〜 255 の整数値
- 名前ラベルをワールド作成モードで設定可能（管理用）
- 初期値: 0

**プレイヤー状態:**

- 各プレイヤーにつき最大 4 種類
- 各状態: 0 〜 255 の整数値
- ルームに入場した時点で全て 0 にリセット

**同期:**

- ワールド状態・プレイヤー状態とも全プレイヤーに同期
- **ルームオーナー（最初に入場したプレイヤー）がマスター**

### 9.2 タイマー仕様

- 1 ワールドにつき最大 5 つのタイマー
- カウント方式: ミリ秒精度でカウントアップ
- 操作: 開始 / 停止 / リセット（0 に戻す）
- **同期方式**: タイマー値は逐次ブロードキャストしない。`{ startTimestamp, isRunning, elapsedAtStop }` の形式で同期し、各クライアントがローカルで現在値を計算する
  - 操作（開始 / 停止 / リセット）が発生したときのみ同期パケットを送信（常時ブロードキャスト不要）
  - ルームオーナーがマスター
- 「タイマーが特定の時間になった瞬間」イベント: 各クライアントがローカル計算値を監視して発火。ルームオーナーが権威的に発火を確認してから全クライアントにブロードキャスト

**オーナー交代時のタイマー引き継ぎ:**

- 各クライアントは同一の `{ startTimestamp, isRunning, elapsedAtStop }` から現在値をローカル計算しているため、全員のローカルタイマーは常に同一の値を示している
- オーナーが退室（クリーンな退室・突然の切断いずれも同様）した場合、新オーナーは追加の同期なしに自分のローカルタイマー状態をそのまま引き継ぐ
- `isRunning=true` のままオーナーが切断した場合も、新オーナーは継続中として扱い、タイマーイベント検知の責任のみ引き継ぐ

### 9.3 インベントリ（オブジェクト保有）

- プレイヤーは 1 種別のオブジェクトを 1 個まで保有できる
- 「オブジェクトを持つ」反応時: ワールド上の対象オブジェクトが消え、プレイヤーのインベントリに入る
- 「特定のオブジェクトを持っているか」条件: インベントリ内容を参照
- ルーム終了時またはプレイヤー退出時: インベントリのオブジェクトはワールド初期配置に戻る
- 同時保有は 1 種別のみ（新たに「持つ」反応が発火した場合は既存のアイテムを元の位置に返す）

### 9.4 ギミックルール構成

1 つのルールは以下の要素で構成される:

```
ルール = 入力イベント（1〜20個・OR結合）+ 条件（0〜20個・AND結合）+ アクション（1〜20個・順番に実行）
```

- 入力イベントが複数ある場合、いずれか 1 つが発火すればルールが起動する（OR 結合）
- 条件が複数ある場合、すべてを満たすときのみアクションを実行する（AND 結合）
- アクションが複数ある場合、定義順にすべてを実行する
- 同一フレームに複数のルールが発火した場合は定義順に実行する
- ルールの評価はルームオーナーが行い、結果を全クライアントに送信する
- ルール・グループの合計数: ワールドで最大 **100 個**

### 9.5 入力イベント

| イベント | 発火タイミング | 相手プレイヤー |
|---|---|---|
| ルーム開始時 | アプリ側のすべての初期化完了後に 1 回発火 | なし |
| 人数変動時 | プレイヤーがルームに入室または退室したとき | なし |
| プレイヤーがオブジェクトに接触 | プレイヤーの物理コライダーが対象オブジェクトのコライダーに触れたとき | なし |
| オブジェクトタップ | プレイヤーが対象オブジェクトをタップ（タッチ判定コライダー使用）したとき | なし |
| エリア侵入 | プレイヤーが対象エリアオブジェクト内に入ったとき | なし |
| タイマーが特定の時間になった瞬間 | 指定タイマーが指定秒数に到達した瞬間 | なし |
| アクションボタンを押す | プレイヤーが画面上のアクションボタンをタップしたとき | なし |
| プレイヤー同士の接触 | プレイヤーが他のプレイヤーに触れたとき | 接触した相手プレイヤー |
| リスポーン | プレイヤーがリスポーン地点（スポーン位置コライダー）の外に出たとき | なし |
| ルーム内ポータル利用 | プレイヤーがルーム内出口ポータルを通ったとき | なし |

アクションボタン: ワールドモード中に表示。

> **相手プレイヤーについて**: 一部のイベントおよび条件は「相手プレイヤー」を確定する（上表の「相手プレイヤー」列）。相手プレイヤーが存在するとき、アクションや条件の対象プレイヤーを「入力プレイヤー」「相手プレイヤー」「全員」から選択できる。相手プレイヤーが存在しない場合は「入力プレイヤー」「全員」のみ選択可。

### 9.6 条件（AND 結合）

**比較演算の種類:**

| 演算 | 説明 |
|---|---|
| 大きい / 小さい / 等しい | 通常の大小・等値比較 |
| X で割った余りが Y | `値 mod X = Y`（X は 2 以上の整数） |

**比較値の参照種別（条件の閾値として指定できる値）:**

| 種別 | 説明 |
|---|---|
| 固定値 | 整数を直接入力 |
| ワールドステートの値 | 指定ステート番号の現在値 |
| イベントに関わったプレイヤーのステートの値 | 入力プレイヤー / 相手プレイヤー（存在するとき）のいずれかを選択し、その指定ステートの値 |

**条件一覧:**

| 条件 | 説明 |
|---|---|
| ワールド状態の値比較 | 指定ワールドステートが比較演算を満たす |
| イベントに関わったプレイヤーの状態比較 | 対象プレイヤー（選択）の指定ステートが比較演算を満たす。または「全プレイヤー中 X 番目に大きい / 小さい」の順位条件も選択可 |
| プレイヤー数条件 | 現在のルーム人数が比較演算を満たす |
| プレイヤー番号条件 | 対象プレイヤー（選択）のルーム参加順番号（1 から）が比較演算を満たす |
| 特定のオブジェクトを持っているか | 対象プレイヤー（選択）のインベントリに指定オブジェクトがあるか |
| プレイヤー同士が重なっている | 入力プレイヤーが他のプレイヤーのコライダーと重なっているとき true。重なっている相手が**相手プレイヤー**になる |
| プレイヤー同士の距離 | 入力プレイヤーから最も近い他プレイヤーまでの距離が X グリッド以内のとき true。その相手が**相手プレイヤー**になる（X は正の小数を含む実数・1 グリッド = 0.5m） |
| プレイヤーから真っすぐ見える | 入力プレイヤーの正面方向に距離 X グリッド（0 < X ≤ 126）のレイを飛ばし、最初に当たった可視物体が他のプレイヤーのとき true。そのプレイヤーが**相手プレイヤー**になる（X は正の小数を含む実数・1 グリッド = 0.5m） |

### 9.7 アクション — 状態変更

**アクションの値の参照種別（セット / 加算 / 減算 で指定できる値）:**

| 種別 | 説明 |
|---|---|
| 固定値 | 整数を直接指定 |
| ワールドステートの値 | 指定ステート番号の現在値 |
| イベントに関わったプレイヤーのステートの値 | 入力プレイヤー / 相手プレイヤー（存在するとき）のいずれかを選択し、その指定ステートの値 |
| 全プレイヤーの特定のステートの合計値 | 全プレイヤーの指定ステートを合計した値 |
| 範囲乱数 | 最小値〜最大値（整数）を指定。ルームオーナーが生成して全プレイヤーに共有する |

| アクション | 説明 |
|---|---|
| ワールド状態を変更 | 指定ワールドステートに値をセット / 加算 / 減算 |
| イベントに関わったプレイヤーの状態を変更 | 対象プレイヤー（入力プレイヤー / 相手プレイヤー ※存在するとき / 全員）の指定ステートに値をセット / 加算 / 減算 |
| タイマー操作 | 指定タイマーを 開始 / 停止 / リセット |

**オーバーフロー・アンダーフロー時の動作:**

ステート値の範囲は 0〜255 であり、加算・減算・セットの結果がこの範囲を超えた場合はクランプする。

- 255 を超えた場合 → 255 に丸める
- 0 を下回った場合 → 0 に丸める
- セットで範囲外の固定値が指定された場合も同様にクランプする

### 9.8 アクション — オブジェクト反応

> 「対象プレイヤー（選択）」と記載されている箇所では、入力プレイヤー / 相手プレイヤー（相手プレイヤーが存在するとき）/ 全員 から選択する。

| アクション | 説明 |
|---|---|
| オブジェクトの表示・非表示切り替え | 対象オブジェクトを表示または非表示にする。カウント・コストは表示時のみ計算 |
| オブジェクトの種類切り替え | 対象オブジェクトを別の種類に入れ替える。カウント・コストは切り替え前後の両方を合算 |
| 文字メッセージ表示 | テキストをポップアップ表示。対象プレイヤー（選択）。1 言語・1 ルールあたり 80 文字以内。対応言語: `docs/screens-and-modes.md` セクション 10 準拠。言語が未設定のユーザーには設定済み言語にフォールバック（英語優先）。UI 詳細: `docs/screens-and-modes.md` セクション 11.7.4 参照 |
| オブジェクトを持つ | 対象プレイヤー（選択）のインベントリに対象オブジェクトを移す（ワールドから消える） |
| 音・音楽を鳴らす | アプリが提供する内蔵サウンドライブラリから選択して再生。設定: ループ ON/OFF・音量（0〜100%）・ピッチ（0.5x〜2.0x）・再生速度（0.5x〜2.0x）。**将来拡張:** 運営がイベント用ワールド専用音源を配信する仕組みの拡張性を確保。プレミアムユーザーが作成時にカスタム音源を設定できる拡張性を確保 |
| オブジェクト移動（直線） | 対象オブジェクトを指定グリッド座標へ指定速度で直線移動させる。移動後の位置が新しい基準位置となる。移動アクション発火時にオーナーが全クライアントへ `{ objectId, fromPos, toPos, speed, moveStartTimestamp }` を送信し、各クライアントがローカルで現在位置を補間する |
| プレイヤー移動 | 対象プレイヤー（選択）を指定したルーム内出口ポータルにワープさせる |
| 状態リセット | 対象（入力プレイヤー / 相手プレイヤー ※存在するとき / 全プレイヤー / ワールド / すべて）のステートを初期値にリセットする |
| エフェクト再生 | 対象プレイヤー（選択）から指定エフェクト（発光等の軽量・短時間パーティクル）を再生する |

### 9.9 入室時のギミック状態同期

プレイヤーがルームに入室（初回入室・再接続後の再入室いずれも同様）した際、オーナーが以下のスナップショットを一括送信する。

| 対象 | 送信内容 |
|---|---|
| ワールドステート | 全ステート（最大 10 種）の現在値 |
| プレイヤーステート | 全プレイヤー分のステート（各最大 4 種）の現在値 |
| タイマー状態 | 全タイマー（最大 5 つ）の `{ startTimestamp, isRunning, elapsedAtStop }` |
| 進行中の移動アクション | 移動完了前のオブジェクト分の `{ objectId, fromPos, toPos, speed, moveStartTimestamp }` |

- 移動完了済みのオブジェクトはワールドの基準位置として保持されているため、別途送信不要
- 入室者はスナップショット受信後、タイマーおよびオブジェクト位置をローカルで再計算して状態を再現する

### 9.10 無限ループ防止

- 1 回のイベント発火から連鎖するアクション実行数が **100 を超えた場合**、無限ループとみなす
- 対処: 全プレイヤーにエラーメッセージを表示し、ルームを初期状態にリセットする

## 11. 公開条件・サムネイル

### 12.1 公開必須条件

ワールドを「公開（Public）」にするために必要な条件:

1. ワールド名が設定されている
2. サムネイルが設定されている
3. スポーン位置が正確に 1 つ配置されている（かつ他オブジェクト・地形と重複していない）
4. 全ての入口ポータル（白）に出口ポータルが設定されている（かつ全ポータルが他オブジェクト・地形と重複していない）

非公開（フレンドのみルームのみ）の場合は上記は不要。

### 12.2 サムネイル

- 縦長（ポートレート比率）
- **設定方法:**
  1. ワールド作成モード内の撮影機能（`docs/screens-and-modes.md` セクション 2.7 準拠）でゲーム内撮影・文字入れして設定
  2. 端末ギャラリーからカスタム画像をアップロード
- サムネイルはサーバー側で適切なサイズに縮小して保存（オリジナルサイズは保存しない）

## 13. マイオブジェクト コライダー編集

### 13.1 概要

マイオブジェクト（ユーザーアップロードのみ）はタイプレベルのコライダーサイズを後から調整できる。テクスチャ編集画面にコライダータブを追加し、タブ切り替えでコライダー編集画面へ遷移する。

### 13.2 アクセス方法

```
ワールドエディタ（配置済みマイオブジェクト選択）→ テクスチャ編集ボタン → コライダータブ
マイオブジェクト管理画面 → テクスチャ編集 → コライダータブ
```

- **変更はタイプレベル**（そのマイオブジェクトの全ワールド・全インスタンスに影響する）
- インスタンス単位の調整はワールドエディタの拡大縮小モードで行う

### 13.3 コライダー編集 UI

| 操作 | 内容 |
|---|---|
| W / D / H 各軸の + / − ボタン | 0.25m 単位で増減（最小 0.25m・0 は後述のトグルで設定） |
| 装飾オブジェクトトグル | コライダーあり ↔ なし（W=D=H=0）を切り替える |

- 装飾オブジェクトトグルを「なし → あり」に切り替えたとき: GLB の AABB から算出したデフォルト値（0.25m 単位切り上げ）を初期値として表示
- 変更は保存ボタンでサーバーに送信（マイオブジェクト定義を更新）

### 13.4 デフォルト・ショップオブジェクトとの違い

| オブジェクト種別 | コライダー設定 | タブ表示 |
|---|---|---|
| マイオブジェクト | 自動検出・ユーザー調整可 | コライダータブ表示 |
| デフォルトオブジェクト | 管理画面でのみ設定 | コライダータブ非表示 |
| ショップオブジェクト | 管理画面でのみ設定 | コライダータブ非表示 |

## 14. 環境音設定

### 13.1 概要

ワールド作成者がワールド固有の環境音を設定できる。入場時からループ再生し、ワールドの雰囲気を演出する。

### 13.2 サウンドライブラリ

内蔵サウンドライブラリから 1 種類を選択する（複数同時再生は不可）。「なし」を選択した場合は環境音なし。

| soundId | 説明 |
|---|---|
| `none` | なし（デフォルト） |
| `forest` | 森・鳥のさえずり |
| `rain` | 雨音 |
| `ocean` | 波音・海辺 |
| `wind` | 風音 |
| `city` | 街のざわめき |
| `cave` | 洞窟・残響 |
| `night` | 夜・虫の声 |

**将来拡張:** 運営がイベント用ワールド専用環境音を追加配信する拡張性を確保（ギミックサウンドと同方針）。

### 13.3 音量設定

- ワールド作成者が 0〜100%（整数）で設定（デフォルト: 100）
- ユーザーの「ワールド効果音」音量設定に対する相対値として乗算される
- 例: 作成者設定 80% × ユーザー設定 50% = 実効音量 40%

### 13.4 再生仕様

- 入場時: 0 からフェードイン（約 2 秒）
- 退場時: フェードアウト（約 1 秒）
- ループ再生（シームレス）
- `WorldSFX` AudioMixer グループを通じて出力

## 12. データ保存フォーマット

### 12.1 保存内容

| データ | 形式 | 説明 |
|---|---|---|
| ワールド定義 JSON | JSON | オブジェクト配置・ギミックルール・背景設定・UV マッピング情報・タグ・地形パレット |
| オブジェクトアトラステクスチャ | PNG | 1024 × 1024、使用オブジェクトテクスチャをパックしたもの |
| 地形アトラステクスチャ | PNG | 1024 × 1024、使用地形テクスチャをパックしたもの（セクション 15.7 参照） |
| 地形ボクセルデータ | バイナリ | チャンクベースの RLE 圧縮ボクセルデータ（セクション 15.13 参照） |
| 背景テクスチャ（任意） | PNG | 1024 × 1024 のカスタム背景 |
| サムネイル | PNG | 縦長、縮小済み |

### 12.2 ワールド定義 JSON 構造（概略）

```json
{
  "version": 2,
  "worldName": "string",
  "tags": ["string"],
  "maxPlayers": 6,
  "background": {
    "type": "solid | gradient | texture",
    "colors": ["#RRGGBB", "#RRGGBB"]
  },
  "ambientSound": {
    "soundId": "none",
    "volume": 100
  },
  "ambientColor": "#FFFFFF",
  "objects": [
    {
      "instanceId": "uuid",
      "objectTypeId": "string",
      "savedVariantId": "string | null",
      "position": { "x": 0.0, "y": 0.0, "z": 0.0 },
      "rotationY": 0,
      "visible": true,
      "size": { "x": 1.0, "y": 1.0, "z": 1.0 }
    }
  ],
  "worldObjectCustomizations": {
    "objectTypeId": {
      "layerJsonUrl": "string",
      "integratedTextureUrl": "string",
      "textureHash": "string"
    }
  },
  "specialObjects": {
    "spawn": { "position": { "x": 0.0, "y": 0.0, "z": 0.0 }, "rotationY": 0 },
    "portals": [...],
    "worldPortals": [...],
    "areas": [...]
  },
  "gimmicks": [
    {
      "ruleId": "uuid",
      "trigger": { "type": "string", "targetId": "string" },
      "conditions": [ { "type": "string", "stateIndex": 0, "operator": "eq|gt|lt", "value": 0 } ],
      "actions": [ { "type": "string", "targetId": "string", "params": {} } ]
    }
  ],
  "worldStates": [
    { "index": 0, "label": "string", "initialValue": 0 }
  ],
  "timers": [
    { "index": 0, "label": "string" }
  ],
  "atlasUVMap": {
    "objectTypeId_or_savedVariantId": { "u": 0.0, "v": 0.0, "w": 0.0, "h": 0.0 }
  },
  "terrain": {
    "palette": ["uid_string_0", "uid_string_1"],
    "voxelDataUrl": "https://cdn.example.com/{sha256}.bin",
    "terrainAtlasUrl": "https://cdn.example.com/{sha256}.png",
    "terrainAtlasUVMap": {
      "uid_string_0": { "u": 0.0, "v": 0.0, "w": 0.25, "h": 0.25 }
    }
  }
}
```

- `maxPlayers`: ルームの最大収容人数（デフォルト 6。通常ユーザー: 2〜6 の整数、プレミアム: 2〜24 の整数）
- `objects[].savedVariantId`: 保存バリアントを使う場合に指定。`null` の場合は `worldObjectCustomizations` のカスタマイズ（なければデフォルト）を使用
- `objects[].size`: インスタンスごとのサイズ上書き（W / D / H、0.25m 単位）。省略（`null`）の場合はオブジェクトタイプの規定サイズを使用
- `worldObjectCustomizations`: このワールドのオブジェクト種別ごとのスコープカスタマイズ（ワールド保存時に更新）
- `atlasUVMap` のキー: objectTypeId（ワールドスコープカスタマイズまたはデフォルト）または savedVariantId（保存バリアント）のいずれか
- `terrain.palette`: 使用する地形種別の UID（int64 を文字列化）の配列（最大 16 要素）
- `terrain.voxelDataUrl`: チャンクボクセルデータのバイナリ URL（content-addressed、キャッシュ可）
- `terrain.terrainAtlasUrl`: 地形テクスチャアトラスの PNG URL
- `terrain.terrainAtlasUVMap`: パレット UID ごとのアトラス UV 位置（u, v: 左下座標、w, h: 幅・高さ、0.0〜1.0）
- `ambientColor`: ワールドの環境カラー（`#RRGGBB`。デフォルト `"#FFFFFF"`。HSV V ≥ 0.25 の制約あり）

### 12.3 ランタイム読み込みフロー

1. ワールド定義 JSON を取得
2. オブジェクトアトラステクスチャ PNG を取得
3. 使用オブジェクトの GLB をオブジェクト ID ごとに取得（クライアント側キャッシュあり）
   - objectTypeId のデフォルト GLB（メッシュ・コライダー）
   - savedVariantId は対応する objectTypeId の GLB を使用（メッシュ・コライダーはベース種別と共通）
4. オブジェクト GLB のメッシュに atlasUVMap を適用して配置・インスタンス化
   - savedVariantId がある場合: そのバリアントのUVを適用
   - ない場合: worldObjectCustomizations のカスタマイズUV（なければデフォルトUV）を適用
5. 地形パレット UID ごとにテクスチャデータを取得（API: UID 単位・クライアントキャッシュあり）
6. 地形アトラステクスチャ PNG を取得
7. 地形ボクセルデータ（バイナリ）を取得してチャンク展開
8. チャンクごとに動的メッシュ・コライダーを生成（詳細: セクション 15.14, 15.15 参照）
9. ギミックエンジンを初期化し、ワールド状態・タイマーを同期開始

## 15. 地形システム

### 15.1 概要

地形はワールドを構成する 2 要素のひとつ（もうひとつはオブジェクト）。キューブ単位で配置し、隣接するキューブに応じて動的にメッシュを生成する。

アバター・アクセサリ・ワールドオブジェクトと同様に、地形テクスチャにも透明ピクセルを使用できる。テクスチャ編集時にベースレイヤーを編集することで不透明・透明を制御する（詳細: `docs/screens-and-modes.md` セクション 21.9 参照）。

### 15.2 種類制限

- 1 ワールドで使用できる地形の種類は最大 **16 種類**
- 同種の地形は個数制限なし

### 15.3 ブロックサイズと配置

- 1 ブロック = 1 グリッド = **0.5m × 0.5m × 0.5m** の立方体
- オブジェクトと共通の 0.5m グリッドに従って配置する

### 15.4 テクスチャ種別

地形テクスチャには 2 種類がある。いずれも全テクスチャ設定は **Filter Mode: Point (no filter)**（ドット感を維持するため）。

| 種別 | テクスチャサイズ | 用途 |
|---|---|---|
| ランダム地形テクスチャ | 256 × 256 | 面ごとに複数バリアントをランダムに選択 |
| 固定地形テクスチャ | 32 × 256 | 面ごとに 1 種のみ（バリアント選択なし） |

### 15.5 ランダム地形テクスチャの構造

256 × 256 のテクスチャを **8 × 8** に分割し、各領域は **32 × 32** ピクセル。

**領域名（左下から X 方向に並ぶ順）:**

```
Y行（下から上）:
行 0（最下行）: 下面 0〜3 / 将来拡張用
行 1:           坂側面下端 0〜3 / 坂側面 0〜3
行 2:           側面上端下端 0〜7
行 3:           側面下端 0〜7
行 4:           側面 0〜7
行 5:           側面上端 0〜7
行 6:           上面中間 0〜7
行 7（最上行）: 上面 0〜7
```

各行の X 方向の並び（列 0〜7）:

| 行 | 列 0〜3 | 列 4〜7 |
|---|---|---|
| 行 0（下面） | 下面 0〜3 | 将来拡張用 |
| 行 1（坂側面） | 坂側面下端 0〜3 | 坂側面 0〜3 |
| その他の行 | 面名 0〜3 | 面名 4〜7（後半） |

### 15.6 固定地形テクスチャの構造

64 × 256 のテクスチャを **2 × 8** に分割し、各領域は **32 × 32** ピクセル。

**領域名（下から順、列 0 が左・列 1 が右）:**

| 行（下から） | 列 0 | 列 1 |
|---|---|---|
| 行 0 | 下面 | 将来拡張用 |
| 行 1 | 坂側面 | 坂側面下端 |
| 行 2 | 側面上端下端 | 将来拡張用 |
| 行 3 | 側面下端 | 将来拡張用 |
| 行 4 | 側面 | 将来拡張用 |
| 行 5 | 側面上端 | 将来拡張用 |
| 行 6 | 上面中間 | 将来拡張用 |
| 行 7 | 上面 | 将来拡張用 |

バリアントが 1 種のみのためインデックス選択は不要。坂側面と坂側面下端はランダム地形テクスチャと同じ行（行 1）に列違いで配置する。

### 15.7 テクスチャアトラス

ワールド保存時にサーバー側で地形テクスチャをアトラス化する（オブジェクトアトラスとは別の **1024 × 1024** テクスチャ）。

- 使用中の地形テクスチャをランダム地形テクスチャ → 固定地形テクスチャの順に貪欲パックする
- 最大 16 種類すべてがランダム地形テクスチャであっても 1024 × 1024 に収まる
  - 256 × 256 が 16 枚: 4 列 × 4 行 = ちょうど 1024 × 1024
  - 固定地形テクスチャ（64 × 256）を混在させると空き領域が生まれるため必ず収まる

### 15.8 面ごとのテクスチャ選択ルール

各ブロックの面は隣接ブロックの有無によってテクスチャ領域が決まる。「同じ種類の地形」が隣接するかどうかで判定する。

#### 上面（坂地形の斜め上面を含む）

| 条件 | 使用領域 |
|---|---|
| 上に同種の地形がない | 上面 0〜7 |
| 上に同種の地形がある | 上面中間 0〜7（Height Culling によって直上ブロックが非表示になったときのみ描画。セクション 15.11 参照） |

#### 側面（坂地形の斜め側面を含む）

| 条件 | 使用領域 |
|---|---|
| 上に同種なし・下に同種あり | 側面上端 0〜7 |
| 上に同種あり・下に同種あり | 側面 0〜7 |
| 上に同種あり・下に同種なし | 側面下端 0〜7 |
| 上に同種なし・下に同種なし | 側面上端下端 0〜7 |

#### 坂地形の側面（三角形面）

| 条件 | 使用領域 |
|---|---|
| 下に同種の地形がある | 坂側面 0〜3 |
| 下に同種の地形がない | 坂側面下端 0〜3 |

#### 下面

常に **下面 0〜3** から選択する。

### 15.9 テクスチャ選択アルゴリズム（ランダム地形テクスチャ）

複数バリアントがある面に対し、グリッド座標・面方向のハッシュ値でインデックスを決定論的に選択する。

**変数:**
- `Xg, Yg, Zg`: グリッド座標（Unity ワールド座標 `Xu, Yu, Zu` との関係: `Xu = Xg × 0.5` 等）
- `directionIndex`: 面の方向ごとに割り当てた正の整数インデックス（実装時に定義）
- `textureTypeLength`: バリアント総数（必ず 2 の累乗: 4 または 8）

**ハッシュ関数（C# 擬似コード）:**

```csharp
uint h = (uint)(73856093 * Xg ^ 19349663 * Yg ^ 83492791 * Zg ^ directionIndex) * 2654435761u;
h ^= h >> 13;
h *= 1274126177u;
int textureIndex = (int)(h & (uint)(textureTypeLength - 1));
```

`textureTypeLength` が 2 の累乗であるため `(textureTypeLength - 1)` は下位ビットがすべて 1 となる。AND 演算で `[0, textureTypeLength - 1]` の範囲に収まることが保証される。

### 15.10 UV 設定

- 1 テクスチャ領域を `[0, 0]`〜`[1, 1]` としたとき、**実際に使用する UV 範囲は `[0.005, 0.005]`〜`[0.995, 0.995]`** とする
  - 端ギリギリを使用すると隣接領域の色がにじみ出るため
  - 実際の UV はアトラス座標変換後にこの範囲がマッピングされる
- **坂地形の三角形側面**: 上記正方形領域の左下（原点に近い側）の **直角三角形** の領域を UV として使用する

### 15.11 プレイヤー高さによる非表示（Height Culling）

#### 概要

プレイヤーの現在位置の真上に不可視のレイを飛ばし、**最初に当たった地形ブロックの高さ以上**にある地形・オブジェクト・他プレイヤーをすべて非表示にする。複数階構造の建物でプレイヤーが自分のいる階を常に見渡せるようにするための機能。

#### デフォルト設定・変更方法

- デフォルト: **ON**（すべてのワールドで有効）
- ワールド作成者がワールドエディタの設定タブからOFFに変更できる（`docs/screens-and-modes.md` セクション 11.7.5 参照）

#### 非表示ルール

1. プレイヤーの位置の真上にレイを飛ばす
2. 最初にヒットした地形ブロックの **Y グリッドインデックス** を「非表示高さ閾値」とする
3. 非表示高さ閾値 **以上** の Y 座標にある地形ブロック・オブジェクト・他プレイヤーをすべて非表示にする
4. ヒットする地形がない場合は何も非表示にしない（屋外）

#### 上面中間テクスチャとの関係

通常、直上に同種の地形がある場合は上面を描画しないが、Height Culling によって直上の地形ブロックが非表示になったとき、その下のブロックの上面は描画する必要がある。このとき上面には **上面中間テクスチャ** を使用する（屋外の上面とは異なるテクスチャを意図的に使い分けられる）。

#### 注意事項

- この非表示処理はクライアントサイドのみ（他プレイヤーの表示には影響しない）
- ワールド編集モード中は常に **OFF**（プレイヤーが存在しないため）

### 15.12 地形ブロック形状

各ブロックは以下の形状（Shape）を持つ。形状は 4 ビットで表現する。

| 値 | 形状 | 説明 |
|---|---|---|
| 0 | empty | 空（地形なし） |
| 1 | cube | 通常の立方体 |
| 2 | ramp_N | 坂（North 側が高く South 側が低い） |
| 3 | ramp_E | 坂（East 側が高く West 側が低い） |
| 4 | ramp_S | 坂（South 側が高く North 側が低い） |
| 5 | ramp_W | 坂（West 側が高く East 側が低い） |
| 6 | diag_NW | 斜め（\ 方向、NW 半分が solid） |
| 7 | diag_SE | 斜め（\ 方向、SE 半分が solid） |
| 8 | diag_NE | 斜め（/ 方向、NE 半分が solid） |
| 9 | diag_SW | 斜め（/ 方向、SW 半分が solid） |

**坂（ramp）**: 側面の対角線を結ぶ面で切った楔形。XZ 方向の投影は 1×1 グリッド全体を覆う。
**斜め（diag）**: 上下の面の対角線を結ぶ面で切った三角柱。XZ 方向の投影は 1×1 グリッドの半分（直角三角形）。

#### 隣接判定における形状の扱い

あるブロック A の面を描画するかどうかを、隣のブロック B の存在で判断するとき、B の形状に応じて以下のように扱う。

| B の形状 | A が B の真下にある（A の上面判定） | A が B の隣にある（A の側面判定） |
|---|---|---|
| cube | ある（上面非表示） | ある（側面非表示） |
| ramp（任意方向） | **ある**（上面非表示） | **ない**（側面表示） |
| diag | **ない**（上面表示） | B の solid 部分が A の面を完全に覆う場合のみ **ある**（それ以外はない） |

diag の側面判定の具体例（diag_NW の場合）:
- A が B の North 側 → B の North 面は full → ある（非表示）
- A が B の West 側 → B の West 面は full → ある（非表示）
- A が B の East 側 → B の East 面は斜め（partial） → ない（表示）
- A が B の South 側 → B の South 面は斜め（partial） → ない（表示）

### 15.13 地形データ構造

#### UID

地形種別には種類ごとに固有の **UID**（ランダムな int64）が割り当てられる。API からは UID 単位でテクスチャデータを取得できる。JSON 表現では 64bit 精度損失を避けるため文字列として扱う。

#### パレット

1 ワールドで使用する地形種別は最大 16 種類。ワールド定義 JSON にパレット（UID の配列）を格納する。パレットインデックスは 0〜15（4 bit に収まる）。

#### ボクセルバイト（1 byte/voxel）

```
bit 7-4: shape nibble（形状。セクション 15.12 の値。0 = empty）
bit 3-0: palette index nibble（0〜15。shape=0 のときは無効）
```

shape nibble が 0 ならそのセルは空。それ以外のときのみ palette index が有効。

#### チャンク構造

- チャンクサイズ: **16 × 16 × 16 ブロック**
  - ワールド 63×31×63 は最大 4×2×4 = **32 チャンク**に分割される（端のチャンクはパディング扱い）
- ボクセル格納順: **X → Z → Y**（X が最内ループ。XZ 方向の繰り返しを RLE で圧縮しやすくするため）
- RLE 圧縮: `(byte_value, count)` のペアを繰り返す。count は 1〜255。同一バイトが 256 以上続く場合は複数ペアに分割
- 空チャンク（全 voxel が empty）はデータを省略する

#### 保存・配信

- パレット（UID 文字列配列）は**ワールド定義 JSON**に格納
- ボクセルデータは**別ファイル**（バイナリ blob）として保存・配信し、ワールド定義 JSON から URL で参照する
- オブジェクトアトラスと同様に content-addressed URL（`{sha256}.bin`）でクライアントキャッシュを利用する

**ボクセルバイナリファイルのフォーマット（概略）:**

```
[4 bytes] magic: 0x4C575654 ("LWVT")
[4 bytes] version: 1
[4 bytes] chunk count
per chunk:
  [1 byte]  cx
  [1 byte]  cy
  [1 byte]  cz
  [2 bytes] RLE data length (bytes)
  [N bytes] RLE data: (value, count) pairs, each 1 byte
```

### 15.14 メッシュ生成ルール

#### 基本方針

- 面ごとに**表面が外に露出している面のみ**メッシュを生成する（非表示面のカリング）
- 下面は**撮影モード時のみ**メッシュを生成する（通常プレイ・編集中は省略）
- 透明テクスチャは**隣接判定に影響しない**（透明地形の隣に地形があっても面は生成しない）

#### チャンク境界

- チャンク境界面の隣接判定では**隣チャンクのボクセルデータ**を参照する
- 隣チャンクが未ロードの場合は「地形あり」として扱う（表示欠けより見えすぎを回避）

#### レンダリング最適化

- 同一地形種別（UID）の同一面種（UV 領域が同じ）をまとめてバッチ化
- 繰り返し出現する面メッシュには `Graphics.DrawMeshInstanced` を使用する
- チャンクごとに再生成可能なメッシュバッファを持つ（隣接変化時は該当チャンクのみ再生成）

#### 透明テクスチャを持つ地形の警告

ベースレイヤーに透明ピクセルを含む地形は、隣接判定には影響しないため内側に描画されない面が発生しうる。ワールドエディタの地形一覧でその種別に **`!`** アイコンを表示し、タップすると「透明領域を含む地形は内側の地形が描画されない場合があります」という注意文を表示する。

### 15.15 コライダー

#### 方針

- `MeshCollider` は使用しない
- 直方体領域を結合して `BoxCollider` を生成する（**グリーディーメッシュ法**）

#### 結合アルゴリズム

1. **Y 層ごとに XZ 平面で走査**し、cube（通常立方体）かつ連続する solid セルを XZ 矩形に結合する（XZ 方向優先）
   - X 方向に延ばせるだけ延ばし、次に Z 方向に延ばす
2. 同一 XZ 矩形が**連続する Y 層でも共通**であれば Y 方向にも結合する
3. すでに結合に使用したセルはスキップする（重複なし）
4. このアルゴリズムは厳密な最適解でなくてよい（XZ 床面が広いケースで効率よく働けば十分）

#### ramp / diag の扱い

ramp・diag ブロックは結合対象から外し、それぞれ個別に**薄い水平 BoxCollider の階段近似**を割り当てる（MeshCollider は使用しない）。

**階段近似の仕様:**

- 各 ramp/diag ブロックに対して、高さ方向を 2〜4 段に分割した薄い水平 BoxCollider を生成する
- 1 段あたりの高さ: 約 0.125〜0.25m（分割数に応じて調整）
- 回転は使用しない（水平 BoxCollider のみ）
- `CharacterController.stepOffset` を **0.26m** に設定し、各 BoxCollider の段差をキャラクターが乗り越えられるようにする

**この方式を選択する理由:**
- MeshCollider（Non-Convex）はモバイルで BVH 構築コストが高く、三角形判定も重い
- 水平 BoxCollider は PhysX の AABB/OBB 判定で O(1) かつ軽量
- 回転 BoxCollider（全体近似）は非スロープ面で phantom wall と通り抜けが発生するため不採用

#### チャンク境界

- コライダーはチャンク内完結を基本とする
- 隣接チャンクにまたがる直方体の結合は、結合コストが低い（チャンク境界で隣を参照するだけで済む）場合のみ実施する

### 15.16 頂点カラーによる簡易アンビエントオクルージョン

地形メッシュの各頂点に対して、隣接ブロックの有無を参照した簡易 AO（Ambient Occlusion）値を頂点カラーとして焼き込む。**計算はメッシュ生成時に 1 度だけ行い、頂点カラーとして保存する**（実行時には再計算しない）。

#### 概要

- 頂点カラーは**無彩色（グレースケール）**
- 隣接ブロックがない状態での頂点カラー明度: **0.75**（ベース暗さ）
- 隣接ブロックが増えるほど黒（明度 0）に近づく
- 最終的な頂点カラー明度: `brightness = 0.75 × clamp01(1 − normalized_darkness)` （明度は 0〜0.75 の範囲）
- **正規化係数: 全グループ共通 2.0**（`normalized_darkness = darkness / 2.0`）

#### 定数定義

ウェイト値は以下の定数で管理し、視覚調整時はここのみ変更する。

```
AO_WEIGHT_STANDARD        = 1.00   // グループ1・3の通常ウェイト
AO_RAMP_HIGH_PRIMARY      = 0.75   // グループ2 高端・グループA主ウェイト
AO_RAMP_HIGH_SECONDARY    = 0.50   // グループ2 高端・グループA副ウェイト（maxのみ採用）
AO_RAMP_HIGH_SIDE         = 0.50   // グループ2 高端・グループB側方ウェイト
AO_RAMP_LOW_PRIMARY       = 0.75   // グループ2 低端・グループA主ウェイト
AO_RAMP_LOW_SECONDARY     = 0.25   // グループ2 低端・グループA副ウェイト（maxのみ採用）
AO_RAMP_LOW_SIDE          = 0.50   // グループ2 低端・グループBウェイト
AO_NORMALIZE              = 2.00   // 正規化係数（全グループ共通）
```

#### 座標系

- +X = East（E）、−X = West（W）
- +Z = North（N）、−Z = South（S）
- +Y = Up

隣接ブロックオフセットは整数グリッド単位 (dx, dy, dz) で表す。

---

#### グループ1: 通常面（cube の全面・ramp の垂直面・diag の垂直面）

**ルール**: 各頂点について「面の2辺方向それぞれ ＋ 法線方向」の合成オフセット先のブロックを参照。ウェイトは各 `AO_WEIGHT_STANDARD`、最大 darkness = 2.0。

**cube 上面（法線 +Y、辺方向 X・Z）:**

| 頂点 | 参照ブロック1 (dx,dy,dz) | 参照ブロック2 (dx,dy,dz) |
|---|---|---|
| NW | (−1, +1,  0) West上  | ( 0, +1, +1) North上 |
| NE | (+1, +1,  0) East上  | ( 0, +1, +1) North上 |
| SW | (−1, +1,  0) West上  | ( 0, +1, −1) South上 |
| SE | (+1, +1,  0) East上  | ( 0, +1, −1) South上 |

**cube East面（法線 +X、辺方向 Y・Z）:**

| 頂点 | 参照ブロック1 (dx,dy,dz) | 参照ブロック2 (dx,dy,dz) |
|---|---|---|
| NE_top | (+1, +1,  0) East上  | (+1,  0, +1) East North |
| SE_top | (+1, +1,  0) East上  | (+1,  0, −1) East South |
| NE_bot | (+1, −1,  0) East下  | (+1,  0, +1) East North |
| SE_bot | (+1, −1,  0) East下  | (+1,  0, −1) East South |

他の側面（West・North・South）は East 面と同じ構造で方向を対称置換して導出する。

**cube 下面（法線 −Y、辺方向 X・Z）:**

| 頂点 | 参照ブロック1 (dx,dy,dz) | 参照ブロック2 (dx,dy,dz) |
|---|---|---|
| NW | (−1, −1,  0) West下  | ( 0, −1, +1) North下 |
| NE | (+1, −1,  0) East下  | ( 0, −1, +1) North下 |
| SW | (−1, −1,  0) West下  | ( 0, −1, −1) South下 |
| SE | (+1, −1,  0) East下  | ( 0, −1, −1) South下 |

> 下面は通常プレイ・編集中はメッシュを生成しないが、AO 値はメッシュ生成時に事前計算して頂点カラーとして保持しておく。撮影モードで下面メッシュを生成する際にその値を適用する。

**ramp・diag の垂直面**: cube の対応する側面と同じルールで計算する（三角形面は頂点数が 3 になるだけでルールは同一）。

---

#### グループ2: ramp の斜め面（4 方向）

各方向の ramp の高端・低端で参照ブロックが異なる。参照は「グループ A（主方向）」と「グループ B（側方向）」の2系統に分け、グループ A 内に複数のブロックがある場合は大きいウェイトを採用（加算しない）。

`darkness = max(グループA 存在ブロックのウェイト) + (グループB ブロックが存在すれば AO_RAMP_HIGH_SIDE または AO_RAMP_LOW_SIDE)`

**ramp_N（North 端が高い）の斜め面:**

| 頂点 | グループA 参照ブロック | ウェイト | グループB 参照ブロック | ウェイト |
|---|---|---|---|---|
| NW_top（高端・W） | 真上 (0,+1,0) | `AO_RAMP_HIGH_PRIMARY` | West上 (−1,+1,0) | `AO_RAMP_HIGH_SIDE` |
|                  | North上 (0,+1,+1) | `AO_RAMP_HIGH_SECONDARY` | | |
| NE_top（高端・E） | 真上 (0,+1,0) | `AO_RAMP_HIGH_PRIMARY` | East上 (+1,+1,0) | `AO_RAMP_HIGH_SIDE` |
|                  | North上 (0,+1,+1) | `AO_RAMP_HIGH_SECONDARY` | | |
| SW_bot（低端・W） | South (0,0,−1) | `AO_RAMP_LOW_PRIMARY`   | West (−1,0,0) | `AO_RAMP_LOW_SIDE` |
|                  | SouthWest (−1,0,−1) | `AO_RAMP_LOW_SECONDARY` | | |
| SE_bot（低端・E） | South (0,0,−1) | `AO_RAMP_LOW_PRIMARY`   | East (+1,0,0) | `AO_RAMP_LOW_SIDE` |
|                  | SouthEast (+1,0,−1) | `AO_RAMP_LOW_SECONDARY` | | |

ramp_E / ramp_S / ramp_W は ramp_N を 90° 回転して導出する（高端・低端の方向と側方向のオフセットを回転させる）。

#### グループ3: diag の斜め面（4 方向）

diag の斜め面は垂直面（Y 方向に傾きなし）だが、XZ 平面で 45° 傾いている。法線を X 成分と Z 成分に分解し、各成分について「上下方向(Y)との合成ブロック」を参照する。ウェイトは各 `AO_WEIGHT_STANDARD`、最大 darkness = 2.0。

**diag_NW（NW 半分が solid、斜め面の法線 (+1,0,−1)/√2）:**

斜め面頂点（NE-SW 対角線上）:

| 頂点 | 参照ブロック1 (dx,dy,dz) | 参照ブロック2 (dx,dy,dz) |
|---|---|---|
| NE_top (+0.25,+0.25,+0.25) | (+1,+1, 0) East上   | ( 0,+1,−1) South上 |
| SW_top (−0.25,+0.25,−0.25) | (+1,+1, 0) East上   | ( 0,+1,−1) South上 |
| NE_bot (+0.25,−0.25,+0.25) | (+1,−1, 0) East下   | ( 0,−1,−1) South下 |
| SW_bot (−0.25,−0.25,−0.25) | (+1,−1, 0) East下   | ( 0,−1,−1) South下 |

> NE_top と SW_top が同一の参照ブロックになるのは、この面においては「上端全体が同じ隣接空間を向いている」ためであり想定どおり。

diag_SE（SE 半分が solid、法線 (−1,0,+1)/√2）、diag_NE（法線 (−1,0,−1)/√2）、diag_SW（法線 (+1,0,+1)/√2）は diag_NW の法線に応じて X・Z 成分の符号を変えて導出する。

---

#### 実装注意事項

- AO 計算はメッシュ生成（面追加）時にまとめて行う
- 頂点カラーは `Color` 型（R=G=B=brightness, A=1.0）で設定する
- チャンク再生成時は当該チャンクの全頂点を再計算する
- ウェイト値の調整は定数（`AO_RAMP_HIGH_PRIMARY` 等）のみを変更する

### 15.17 環境カラーと描画合成

ワールドごとに設定できる単色の「環境カラー」を全描画に乗算して適用する。Unlit シェーダーのライティングなし制約を維持しつつ、簡易な雰囲気カラーリングを実現する。

#### 環境カラー仕様

- デフォルト: 白（`#FFFFFF`）
- ワールド設定パネル（セクション 11.7.5）でカラーピッカーから変更可能
- **最低明度制限**: HSV の V 値 ≥ **0.25**（画面が暗すぎるのを防ぐ）
- ワールド定義 JSON に `ambientColor: "#RRGGBB"` として保存する

#### 描画合成ルール

| 描画対象 | 合成式 |
|---|---|
| **地形** | `texel_color × vertex_AO_color × ambient_color` |
| **ワールドオブジェクト** | `texel_color × ambient_color` |
| **アバター・アクセサリ** | `texel_color × ambient_color` |

- `vertex_AO_color`: 面上の位置に応じて各頂点の頂点カラーを補間した値（セクション 15.16）
- `ambient_color`: ワールド設定の環境カラー（Unity シェーダーの `_AmbientColor` プロパティとして渡す）
- 地形用シェーダーと共通 Unlit シェーダー（アバター・オブジェクト用）のいずれも同一の `_AmbientColor` uniform を参照する
